    • Introduction

    Semantic elements are elements that are used in HTML to cleary describe its meaning to both the browser and the developer. They a better level of readability than their non-semantic counterparts <div> and <span> thus making it easier to edit the HTML after a long period of time.

    Header Element

    <header></header>

    Defines the introductory content. This should be used to wrap content, such as a your headings for a <section> or <article>. Note that there can be more than one <header> element within a single document.

    Main Element

    <main></main>

    Defines the main content of the document. This area will be where the primary focus of the document will be placed. There may be several <article> or <section> elements nested in here as well depending on what kind of document it is.

    Section Element

    <section></section>

    Defines a section within a document. this element should be used to wrap related content.

    Article Element

    <article></article>

    Defines independant content. Depending on the document, there may be several articles within a section or vice-versa. This element is commonly used for:

    • Blog posts
    • Forum posts
    • Newspaper articles

    Aside Element

    <aside></aside>

    Defines content aside from what is around it. Commonly used as a sidebar and should be relevant to what it is next to.

    Nav Element

    <nav></nav>

    This element is used to wrap your navigational links, common ones being:

    • Home
    • About
    • Contact

    Footer Element

    <footer></footer>

    Defines the final content for the element it is nested with in. There may be more than one <Footer> element within a single document. Common content found in this element include:

    • Author
    • Address
    • Contact info
